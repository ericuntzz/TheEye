BLOCKING ISSUES (CRITICAL -- fix before next deployment)

  None -- no blocking code issues found.

  INFRASTRUCTURE NOTE: The Next.js 15.5.12 dev server has a severe
  webpack chunk-splitting bug. When multiple routes compile concurrently,
  webpack generates _document.js with references to chunk files (611.js,
  996.js) that are never written to disk, causing cascading 500 errors.
  The routes-manifest.json is deleted during recompilation and never
  regenerated. Production build (next build + next start) is unaffected.

  Workaround: Start dev server, wait >10s before first request, and
  hit routes one at a time with 3+ second pauses between requests.
  This is fragile and developer-hostile.

  Recommended: Investigate adding serverExternalPackages for jsfeat
  and onnxruntime-node to next.config.ts to reduce webpack chunk
  complexity, or upgrading Next.js.
